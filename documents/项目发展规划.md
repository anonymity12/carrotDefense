# 项目发展规划

## 1. 代码库清理与优化
为了保持项目的整洁和可维护性，计划对现有代码库进行以下清理工作：

- **移除冗余脚本**：删除 `scripts` 文件夹及其内容（如 `generatePandaIcons.ts`）。这些脚本主要用于早期的资源生成，在项目进入稳定开发阶段后已不再需要。
- **目录结构规范化**：确保所有源代码、资源文件和文档都存放在标准的目录结构中，便于团队协作和后续维护。

## 2. 新增功能：用户身份体系系统
为了更好地了解用户行为、优化游戏体验以及提供基于位置的服务，计划引入一套完整的用户身份体系。

### 2.1 数据采集核心指标
系统将重点记录以下用户数据：

- **游戏时长 (Playtime)**
  - **目的**：分析用户粘性，识别核心玩家群体，优化游戏节奏。
  - **实现方式**：记录用户每次会话的开始和结束时间，累计总游戏时长。

- **地理位置信息 (GPS Data)**
  - **目的**：分析玩家的地域分布，未来可能开发基于地理位置（LBS）的特殊关卡或活动（例如：在特定立交桥附近玩游戏获得加成）。
  - **实现方式**：在用户授权的前提下，获取移动设备的 GPS 坐标（经纬度）。

- **网络地址 (IP Address)**
  - **目的**：辅助定位，进行安全风控，防止作弊行为。
  - **实现方式**：在与服务器通信时记录客户端 IP 地址。

### 2.2 隐私与合规
- 在采集 GPS 和 IP 等敏感信息前，必须设计清晰的**用户授权弹窗**。
- 确保数据采集符合相关法律法规（如个人信息保护法），并提供用户撤回授权的选项。

## 3. 后续展望
基于上述身份体系，未来可扩展的功能包括：
- **用户排行榜**：基于地区或游戏时长的排行榜。
- **云存档**：将用户进度与身份绑定，实现跨设备同步。
- **个性化推荐**：根据用户习惯推荐关卡或游戏模式。
